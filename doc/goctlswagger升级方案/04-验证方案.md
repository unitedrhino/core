# 04 - 验证方案

## 1. 单元测试

```shell
cd D:\code\company\go-zero\tools\goctl
go test ./api/swagger/... -v -run TestSwagger2ToOpenAPI3
```

验证 `swagger2ToOpenAPI3()` 函数能正确将 Swagger 2.0 转换为 OpenAPI 3.0。

## 2. 端到端测试

用业务项目的 api.api 文件生成 OpenAPI 3.0 文档：

```shell
cd D:\code\company\things\service\apisvr
goctl api swagger -filename swagger.json -api http/api.api -dir ./http
```

## 3. 输出验证清单

生成的 JSON 文件应满足以下条件：

- [ ] 顶层包含 `"openapi": "3.0.x"` 而非 `"swagger": "2.0"`
- [ ] 包含 `servers` 数组而非 `host`/`basePath`/`schemes`
- [ ] 类型定义在 `components.schemas` 下而非 `definitions` 下
- [ ] 所有 `$ref` 路径为 `#/components/schemas/Xxx` 格式
- [ ] POST 请求的 body 参数在 `requestBody` 中而非 `parameters` 中
- [ ] 响应体在 `content.application/json.schema` 中而非直接 `schema`

## 4. 在线验证

将生成的 JSON 粘贴到 Swagger Editor 验证：

1. 打开 https://editor.swagger.io/
2. 粘贴生成的 JSON 内容
3. 确认无解析错误、所有接口可正常展示

## 5. 与 Apifox 参考文件对比

参考文件：`apifox3.0导入示例.json`

对比要点：

- 顶层结构一致（openapi、info、tags、paths、components）
- requestBody 格式一致
- components/schemas 中的 $ref 引用格式一致
- 响应体 content 结构一致
