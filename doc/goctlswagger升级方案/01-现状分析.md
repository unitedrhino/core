# 01 - 现状分析

## 1. goctl swagger 命令架构

goctl 中存在**两个版本**的 swagger 生成命令：

### 旧版命令 `swagger`（当前在用）

- **入口**: `tools/goctl/api/swagger/cmd.go` → `SwaggerCommand()`
- **核心逻辑**: `tools/goctl/api/swagger/generate/` 包
- **数据结构**: 自定义 Swagger 2.0 结构体（`generate/entities.go`）
- **命令注册**: `tools/goctl/api/cmd.go:38`

### 新版命令 `swagger2`（flag 注册被注释，暂不可用）

- **入口**: `tools/goctl/api/swagger/command.go` → `Command()`
- **核心逻辑**: `tools/goctl/api/swagger/swagger.go` → `spec2Swagger()`
- **数据结构**: 使用 `github.com/go-openapi/spec` 库
- **命令注册**: `tools/goctl/api/cmd.go:37`（flag 在 117-120 行被注释）

## 2. 生成流程

```text
.api 文件
  ↓ parser.Parse()
ApiSpec 结构体
  ↓ fillAllStructs() / applyGenerate()
Swagger 2.0 JSON
  ↓ json.MarshalIndent() / json.Encode()
写入文件
```

## 3. 核心文件清单

| 文件 | 职责 |
|------|------|
| `api/swagger/swagger.go` | `spec2Swagger()` 核心转换、类型映射 |
| `api/swagger/cmd.go` | 旧版命令入口 `SwaggerCommand()` |
| `api/swagger/command.go` | 新版命令入口 `Command()` |
| `api/swagger/const.go` | 常量定义，`swaggerVersion = "2.0"` |
| `api/swagger/path.go` | 路径和操作生成 |
| `api/swagger/parameter.go` | 参数生成（header/path/query/body/form） |
| `api/swagger/response.go` | 响应生成 |
| `api/swagger/definition.go` | Schema 定义生成 |
| `api/swagger/properties.go` | 属性生成 |
| `api/swagger/api.go` | `fillAllStructs()` 结构体填充 |
| `api/swagger/generate/` | 旧版生成逻辑（自定义结构体） |
