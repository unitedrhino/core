# oldCore 本地联调环境

本文记录 AI/本机联调 oldCore 后端与 `ykhl-harmony-app` iPhone 真机客户端的启动步骤。

## 目标链路

- 后端只在本机跑 `coresvr`，监听 `0.0.0.0:7777`。
- iPhone 真机访问本机局域网地址：`http://192.168.1.43:7777`。
- 数据库通过 SSH 隧道连接远端真实数据：
  - 本机：`127.0.0.1:13306`
  - 远端：`root@new.ykhl.vip:127.0.0.1:3306`
- Redis、NATS、日志、进程都在本机。
- 默认不部署、不推送、不改远端服务。

## 一键启动后端

```bash
cd /Users/ykhl/Desktop/oldCore
scripts/dev/core-local.sh start
scripts/dev/core-local.sh status
```

正常状态应看到：

```text
coresvr: running pid ...
port 13306: listening
port 6379: listening
port 4222: listening
port 7777: listening
```

验证 debug 接口：

```bash
curl -fsS http://127.0.0.1:7777/api/v1/system/common/debug
curl -fsS http://192.168.1.43:7777/api/v1/system/common/debug
```

## 查看后端日志

```bash
cd /Users/ykhl/Desktop/oldCore
scripts/dev/core-local.sh logs
```

常用筛选：

```bash
tail -f cmd/logs/coresvr.log | rg '192\.168\.1|HTTP|websocket|login|error'
```

## 重启或停止

重启后端和隧道：

```bash
cd /Users/ykhl/Desktop/oldCore
scripts/dev/core-local.sh restart
```

停止后端和 DB 隧道：

```bash
cd /Users/ykhl/Desktop/oldCore
scripts/dev/core-local.sh stop
```

如果也要停本机 Redis/NATS：

```bash
cd /Users/ykhl/Desktop/oldCore
CORE_STOP_INFRA=1 scripts/dev/core-local.sh stop
```

## 启动 iPhone 真机客户端

先确认设备：

```bash
cd /Users/ykhl/Desktop/ykhl-harmony-app
scripts/hx-cli.sh devices list --platform ios-iPhone
```

当前常用设备：

```text
00008110-00084C660C6B801E
```

启动自定义基座：

```bash
cd /Users/ykhl/Desktop/ykhl-harmony-app
scripts/hx-cli.sh launch app-ios \
  --iosTarget device \
  --deviceId 00008110-00084C660C6B801E \
  --playground custom \
  --compile false \
  --continue-on-error true
```

看到类似下面日志表示客户端已启动：

```text
项目 [ykhl-harmony-app] 已启动
开发环境 at config/index.uts
```

## 联调成功判断

后端日志里应看到 iPhone 局域网 IP 请求，例如：

```text
[HTTP] 200 - POST /api/v1/system/user/self/app/read - 192.168.1.x:...
```

当前已验证过的启动请求：

```bash
curl -fsS -X POST http://127.0.0.1:7777/api/v1/system/user/self/app/read \
  -H 'Content-Type: application/json' \
  -d '{"code":"client-app-android"}'
```

正常返回应包含：

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "code": "client-app-android",
    "config": "{...}",
    "tenantCodes": ["default"]
  }
}
```

## 客户端本地环境配置

`/Users/ykhl/Desktop/ykhl-harmony-app/config/index.uts` 已配置：

```text
development -> local
hostUrl -> http://192.168.1.43:7777
wsUrl -> ws://192.168.1.43:7777/api/v1/system/common/websocket/connect
```

如果换 Wi-Fi 或本机 IP 变化，只需要更新 `local.hostUrl` 和 `local.wsUrl`。

查看本机当前局域网 IP：

```bash
ipconfig getifaddr en0
```

## 常见问题

### DB 隧道断了

现象：

```text
port 13306: closed
```

处理：

```bash
cd /Users/ykhl/Desktop/oldCore
scripts/dev/core-local.sh start
```

### iPhone 请求不到本机

先确认本机局域网地址能访问：

```bash
curl -fsS http://192.168.1.43:7777/api/v1/system/common/debug
```

如果不通，检查：

- Mac 和 iPhone 是否在同一个 Wi-Fi。
- 本机 IP 是否还是 `192.168.1.43`。
- `coresvr` 是否监听 `0.0.0.0:7777`。

### 真机提示标准基座无法运行

iOS 真机不能跑未签名标准基座，需要显式使用自定义基座：

```bash
--playground custom
```

### HBuilderX 长时间不退出

`launch app-ios` 可能保持运行态等待日志。确认手机已经启动、后端已有请求后，可以结束该 CLI 进程；后端 `coresvr` 不受影响。

### Docker 不可用

当前机器没有可用 Docker daemon，脚本会自动使用本机 `redis-server` 和 `nats-server` 兜底，不需要手动启动 Docker。
