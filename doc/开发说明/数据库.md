# 数据库使用说明

## 1. 数据类型规范

### 1.1 布尔值处理
- **不直接使用 `bool` 类型**，而是采用 `int` 类型表示
  - `1` 表示 `true`（启用、是、开启）
  - `2` 表示 `false`（禁用、否、关闭）
- 示例：`IsTenantOwner: 1`（是租户所有者）、`Status: 2`（禁用）

### 1.2 字符串处理
- **可能为空的字符串**：使用 `sql.NullString` 类型避免零值问题
  - 应用场景：用户名、邮箱、手机号等可能为空的字段
- **固定长度字符串**：使用 `CHAR` 类型
  - 示例：密码字段 `password CHAR(32)`
- **可变长度字符串**：使用 `VARCHAR` 类型并指定合理长度
  - 示例：用户名 `user_name VARCHAR(200)`、邮箱 `email VARCHAR(255)`

### 1.3 数值类型
- **ID 字段**：使用 `BIGINT` 类型（对应 Go 中的 `int64`）
  - 示例：`user_id BIGINT NOT NULL AUTO_INCREMENT`
- **计数字段**：使用 `BIGINT` 类型
  - 示例：`device_count BIGINT DEFAULT 0`
- **状态字段**：使用 `BIGINT` 或 `SMALLINT` 类型
  - 示例：`status SMALLINT DEFAULT 1`

### 1.4 复杂数据类型
- **数组类型**：使用 `JSON` 类型存储
  - 示例：`sync_dept_ids JSON`（同步部门ID列表）
- **字典类型**：使用 `JSON` 类型存储
  - 示例：`tags JSON`（标签）、`field_map JSON`（字段映射）

## 2. 数据库表设计

### 2.1 表名规范
- 采用**小写+下划线**命名格式
- 系统表以 `sys_` 为前缀
- 示例：`sys_user_info`（用户信息表）、`sys_role_info`（角色信息表）

### 2.2 主键设计
- 通常使用 `id` 或 `xxx_id` 作为主键
- 优先使用自增主键
- 示例：`user_id BIGINT PRIMARY KEY AUTO_INCREMENT`

### 2.3 索引设计
- **唯一索引**：用于唯一约束字段
  - 示例：用户名、邮箱、手机号等
  - 命名格式：`idx_表名_字段名`，如 `idx_sys_user_info_tc_un`
- **复合索引**：优化多字段查询
  - 示例：用户-租户关联表的 `sys_user_tenant_user_tenant` 索引

### 2.4 软删除机制
- 通过 `deleted_time` 字段实现软删除
  - `0` 表示未删除
  - 非零值表示删除时间戳
- 示例：`deleted_time BIGINT DEFAULT 0`

## 3. GORM 使用规范

### 3.1 模型定义
- 使用 `gorm` 标签指定数据库映射关系
- 包含表名方法 `TableName()`
- 示例：
  ```go
  type SysUserInfo struct {
      UserID   int64  `gorm:"column:user_id;primary_key;AUTO_INCREMENT;type:BIGINT;NOT NULL"`
      UserName string `gorm:"column:user_name;uniqueIndex:idx_sys_user_info_tc_un;type:VARCHAR(200)"`
  }
  
  func (m *SysUserInfo) TableName() string {
      return "sys_user_info"
  }
  ```

### 3.2 仓储模式
- 每个实体对应一个 `Repo` 结构体
- 使用 `NewXxxRepo` 函数创建仓储实例
- 提供标准的 CRUD 方法：
  - `Insert`：插入数据
  - `FindOneByFilter`：根据条件查询单条数据
  - `FindByFilter`：根据条件查询多条数据
  - `Update`：更新数据
  - `Delete`：删除数据

### 3.3 查询构建
- 使用 `fmtFilter` 方法统一构建查询条件
- 支持分页查询（`PageInfo`）
- 支持预加载关联数据（`Preload`）
- 示例：
  ```go
  func (p UserInfoRepo) fmtFilter(ctx context.Context, f UserInfoFilter) *gorm.DB {
      db := p.db.WithContext(ctx)
      if f.UserID != 0 {
          db = db.Where("user_id=?", f.UserID)
      }
      if f.WithRoles {
          db = db.Preload("Tenants.Roles").Preload("Tenants.Roles.Role")
      }
      return db
  }
  ```

### 3.4 错误处理
- 使用 `stores.ErrFmt` 统一处理数据库错误
- 示例：
  ```go
  func (p RoleInfoRepo) Insert(ctx context.Context, data *SysRoleInfo) error {
      result := p.db.WithContext(ctx).Create(data)
      return stores.ErrFmt(result.Error)
  }
  ```

## 4. 关联关系处理

### 4.1 一对多关系
- 使用外键关联
- 示例：用户与租户的关系
  ```go
  type SysUserInfo struct {
      UserID  int64            `gorm:"column:user_id;primary_key"`
      Tenants []*SysUserTenant `gorm:"foreignKey:UserID;references:UserID"`
  }
  ```

### 4.2 多对多关系
- 通过中间表实现
- 示例：用户与角色的关系（通过 `sys_user_role` 表）

### 4.3 嵌入结构体
- 使用 `embedded` 标签实现结构体组合
- 可以指定嵌入前缀
- 示例：
  ```go
  type SysConfigInfo struct {
      ID   int64                    `gorm:"column:id;primary_key"`
      Oem  systemConfig.OemConfig   `gorm:"embedded;embeddedPrefix:oem_"`
      Sms  systemConfig.SmsConfig   `gorm:"embedded;embeddedPrefix:sms_"`
  }
  ```

## 5. 避免零值问题的策略

### 5.1 使用 `sql.NullString`
- 处理可能为空的字符串字段
- 避免将空字符串与零值字符串混淆

### 5.2 非零默认值
- 为数值字段设置合理的非零默认值
- 示例：`status BIGINT DEFAULT 1`（默认启用）

### 5.3 使用指针类型
- 对于可选字段使用指针类型
- 示例：`updated_time *stores.Cmp`（更新时间比较条件）

### 5.4 业务逻辑验证
- 在业务层确保必填字段有有效值
- 示例：用户注册时验证用户名、密码等必填字段

## 6. 库表新增方案

在每个服务的 `internal/repo/relationDB` 目录下有 `example.go` 文件，提供了标准的仓库操作模板。新增库表时，请按照以下步骤操作：

### 6.1 生成GORM模型
1. 借助 `https://sql2gorm.mccode.info/` 工具将SQL语句转换为GORM模型
2. 将生成的模型代码放到服务的 `internal/repo/relationDB/model.go` 文件中

### 6.2 创建仓库实现
1. 复制 `internal/repo/relationDB/example.go` 文件到当前目录下
2. 修改文件名和表名，例如：将 `example.go` 重命名为 `user_info.go`
3. 将文件中所有的 `Example` 替换为对应的数据表名称（如 `UserInfo`）
4. 根据业务需求定制修改对应的CRUD函数

### 6.3 示例流程

#### 步骤1：生成GORM模型
将SQL表结构转换为GORM模型并添加到 `model.go` 中：

```go
// model.go
type SysUserInfo struct {
    UserID   int64  `gorm:"column:user_id;primary_key;AUTO_INCREMENT;type:BIGINT;NOT NULL"`
    UserName string `gorm:"column:user_name;uniqueIndex:idx_sys_user_info_tc_un;type:VARCHAR(200)"`
    Status   int    `gorm:"column:status;type:INT;DEFAULT:1"`
    DeletedTime int64 `gorm:"column:deleted_time;type:BIGINT;DEFAULT:0"`
    // 其他字段...
}

func (m *SysUserInfo) TableName() string {
    return "sys_user_info"
}
```

#### 步骤2：创建仓库文件
复制 `example.go` 为 `user_info.go` 并修改，完整示例如下：

```go
// user_info.go
package relationDB

import (
    "context"
    "time"
    "gitee.com/unitedrhino/share/stores"
    "gorm.io/gorm"
    "gorm.io/gorm/clause"
)

type UserInfoRepo struct {
    db *gorm.DB
}

func NewUserInfoRepo(in any) *UserInfoRepo {
    return &UserInfoRepo{db: stores.GetCommonConn(in)}
}

type UserInfoFilter struct {
    UserID  int64
    UserName string
    Status   int
}

func (p UserInfoRepo) fmtFilter(ctx context.Context, f UserInfoFilter) *gorm.DB {
    db := p.db.WithContext(ctx)
    if f.UserID != 0 {
        db = db.Where("user_id = ?", f.UserID)
    }
    if f.UserName != "" {
        db = db.Where("user_name = ?", f.UserName)
    }
    if f.Status != 0 {
        db = db.Where("status = ?", f.Status)
    }
    // 软删除过滤
    db = db.Where("deleted_time = ?", 0)
    return db
}

func (p UserInfoRepo) Insert(ctx context.Context, data *SysUserInfo) error {
    result := p.db.WithContext(ctx).Create(data)
    return stores.ErrFmt(result.Error)
}

func (p UserInfoRepo) FindOneByFilter(ctx context.Context, f UserInfoFilter) (*SysUserInfo, error) {
    var result SysUserInfo
    db := p.fmtFilter(ctx, f)
    err := db.First(&result).Error
    if err != nil {
        return nil, stores.ErrFmt(err)
    }
    return &result, nil
}

func (p UserInfoRepo) FindByFilter(ctx context.Context, f UserInfoFilter, page *stores.PageInfo) ([]*SysUserInfo, error) {
    var results []*SysUserInfo
    db := p.fmtFilter(ctx, f).Model(&SysUserInfo{})
    db = page.ToGorm(db)
    err := db.Find(&results).Error
    if err != nil {
        return nil, stores.ErrFmt(err)
    }
    return results, nil
}

func (p UserInfoRepo) CountByFilter(ctx context.Context, f UserInfoFilter) (size int64, err error) {
    db := p.fmtFilter(ctx, f).Model(&SysUserInfo{})
    err = db.Count(&size).Error
    return size, stores.ErrFmt(err)
}

func (p UserInfoRepo) Update(ctx context.Context, data *SysUserInfo) error {
    err := p.db.WithContext(ctx).Where("user_id = ?", data.UserID).Save(data).Error
    return stores.ErrFmt(err)
}

func (p UserInfoRepo) DeleteByFilter(ctx context.Context, f UserInfoFilter) error {
    // 软删除
    db := p.fmtFilter(ctx, f)
    err := db.Update("deleted_time", time.Now().UnixMilli()).Error
    return stores.ErrFmt(err)
}

func (p UserInfoRepo) Delete(ctx context.Context, id int64) error {
    // 软删除
    err := p.db.WithContext(ctx).Where("user_id = ?", id).Update("deleted_time", time.Now().UnixMilli()).Error
    return stores.ErrFmt(err)
}

func (p UserInfoRepo) FindOne(ctx context.Context, id int64) (*SysUserInfo, error) {
    var result SysUserInfo
    err := p.db.WithContext(ctx).Where("user_id = ? AND deleted_time = ?", id, 0).First(&result).Error
    if err != nil {
        return nil, stores.ErrFmt(err)
    }
    return &result, nil
}

// 批量插入记录
func (p UserInfoRepo) MultiInsert(ctx context.Context, data []*SysUserInfo) error {
    err := p.db.WithContext(ctx).Clauses(clause.OnConflict{UpdateAll: true}).Model(&SysUserInfo{}).Create(data).Error
    return stores.ErrFmt(err)
}

// 批量更新记录
func (d UserInfoRepo) UpdateWithField(ctx context.Context, f UserInfoFilter, updates map[string]any) error {
    db := d.fmtFilter(ctx, f)
    err := db.Model(&SysUserInfo{}).Updates(updates).Error
    return stores.ErrFmt(err)
}
```

### 6.4 注意事项
- 确保模型与数据库表结构保持一致
- 使用 `stores.GetCommonConn(in)` 获取数据库连接
- 所有仓库都应该实现标准的CRUD接口
- 软删除使用 `deleted_time` 字段，避免物理删除
- 使用事务处理复杂的数据库操作
- 合理使用索引优化查询性能
- 使用 `stores.ErrFmt()` 统一处理数据库错误
