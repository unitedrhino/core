# 参数命名风格说明

## 1. 标识符命名规范

### 1.1 结构体命名
- 采用大驼峰命名法（PascalCase）
- 系统模型以 `Sys` 为前缀，如：`SysUserInfo`、`SysRoleInfo`
- 仓储结构体以 `Repo` 为后缀，如：`UserInfoRepo`、`TenantInfoRepo`
- 过滤条件结构体以 `Filter` 为后缀，如：`UserInfoFilter`、`RoleInfoFilter`

### 1.2 字段命名
- 常规字段采用小驼峰命名法（camelCase），如：`userName`、`nickName`
- ID 字段使用大写 `ID` 后缀，如：`UserID`、`TenantID`、`ParentID`
- 布尔类型字段使用 `Is` 前缀，如：`IsTenantOwner`、`IsAuthChildren`、`IsAddSync`
- 计数相关字段使用 `Count` 后缀，如：`UserCount`、`DeviceCount`、`ChildrenCount`
- 时间相关字段使用 `Time` 后缀，如：`UpdatedTime`、`DeletedTime`

### 1.3 变量与函数命名
- 变量名采用小驼峰命名法，如：`userID`、`tenantCode`
- 函数名采用小驼峰命名法，如：`FindOneByFilter`、`Insert`
- 私有函数以下划线开头，如：`fmtFilter`、`accountsFilter`

## 2. 特殊字段命名规则

### 2.1 布尔值字段
- 使用 `Is` 前缀标识布尔含义
- 实际存储使用 `int` 类型，`1` 表示 `true`，`2` 表示 `false`
- 示例：`IsTenantOwner`（是否为租户所有者）、`IsAddSync`（是否自动同步）

### 2.2 状态字段
- 直接命名为 `Status`
- 使用 `int` 类型表示不同状态值
- 示例：`Status: 1`（启用）、`Status: 2`（禁用）

### 2.3 复合含义字段
- 采用驼峰拼接方式表达完整含义
- 示例：`LastIP`（最后登录IP）、`RegIP`（注册IP）、`AllDeviceCount`（所有设备数量）

## 3. 命名一致性原则

### 3.1 跨层级一致性
- 数据库列名与结构体字段名保持语义一致（GORM自动映射）
- API参数与内部结构体字段保持语义一致
- 日志与错误信息中的字段名与代码保持一致

### 3.2 领域术语一致性
- 同一概念在整个系统中使用统一命名
- 示例：租户编码统一使用 `tenantCode`，用户ID统一使用 `userID`

### 3.3 避免歧义
- 使用清晰明确的命名，避免缩写（除非是广为人知的缩写）
- 示例：使用 `email` 而不是 `eml`，使用 `phone` 而不是 `tel`

## 5. Protobuf字段命名规范

### 5.1 字段命名
- 采用小驼峰命名法（camelCase），如：`userName`、`tenantCode`
- 字段编号从1开始，依次递增，不重复
- 避免使用保留字作为字段名

### 5.2 字段类型
- 使用proto3基础类型：`int64`、`string`、`bool`等
- 可选字段使用`google/protobuf/wrappers.proto`类型，如：`google.protobuf.StringValue email`
- 重复字段使用`repeated`关键字，如：`repeated int64 ids`
- 映射字段使用`map`关键字，如：`map<string,string> params`

### 5.3 字段注释
- 每个字段必须添加注释，说明字段含义、用途和约束条件
- 注释使用`//`开头，紧跟字段定义
- 注释应清晰明了，避免歧义

### 5.4 Protobuf字段示例

```proto
// 用户创建请求
message UserInfoCreateReq {
    string userName = 1; // 登录用户名（必填）
    string password = 2; // 密码（必填）
    string phone = 3; // 手机号（可选）
    google.protobuf.StringValue email = 4; // 邮箱（可选）
}

// ID列表
message IDList {
    repeated int64 ids = 1; // ID集合
}

// 通知配置
message NotifyConfig {
    int64 id = 1; // 配置ID编号
    string group = 2; // 通知分组
    string code = 3; // 通知类型编码
    map<string,string> params = 12; // 通知变量属性（key是参数名，value是参数描述）
}
```
